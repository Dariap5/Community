# 📋 Гайд по Админ Панели Telegram Sales Bot

## 🎯 Общая архитектура

### Структура данных
```
Funnel (воронка)
  └─ FunnelStep (шаг воронки)
      ├─ StepMessage (субсообщение) × N
      │   ├── message_type: text|photo|document|video|video_note|voice
      │   ├── content_text: HTML текст с форматированием
      │   ├── content_file: Telegram file_id для медиа
      │   ├── caption: описание для фото/видео
      │   ├── parse_mode: HTML|Markdown (для текста и caption)
      │   └── delay_after_seconds: задержка перед следующим сообщением
      │
      └─ StepButton (кнопка) × N
          ├── text: текст на кнопке
          ├── button_type: url|callback|payment
          └── value: URL или callback_data
```

### Workflow пользователя в боте
1. Пользователь отправляет `/start` или переходит по deeplink
2. Бот проверяет `UserFunnelState` — если активная воронка есть, не запускает новую
3. Бот отправляет сообщения **шага в порядке message_order**
4. После каждого сообщения ждет **delay_after_seconds**:
   - ≤ 60сек: ждет inline (`asyncio.sleep`)
   - > 60сек: создает задачу `ScheduledTask` и возобновляет после задержки
5. После **всех сообщений** показывает кнопки (одну клавиатуру на весь шаг)
6. При клике на кнопку → переход на следующий шаг

## 📝 Редактирование сообщений

### Как добавить сообщение
1. Нажми "Добавить" в секции "Субсообщения"
2. Откроется новое пустое сообщение
3. Заполни:
   - **Тип**: текст/фото/видео/документ/видеозаписка/голос (dropdown)
   - **Текст**: содержимое сообщения (для текста)
   - **File ID**: ID файла в Telegram (для медиа) — см. ниже как получить
   - **Caption**: описание (для фото/видео/голоса)
   - **Parse Mode**: HTML или Markdown (как форматировать текст)
   - **Задержка**: сколько секунд ждать перед следующим сообщением

### Форматирование текста (HTML mode)
```html
<b>Жирный текст</b>
<i>Курсив</i>
<u>Подчеркивание</u>
<s>Зачеркивание</s>
<code>Моноширинный шрифт</code>
<blockquote>Цитата</blockquote>
<a href="https://example.com">Ссылка</a>
```

❌ **НЕ поддерживаются в Telegram:**
- Цвета (`<span style="color:red">`)
- CSS стили
- Размер шрифта
- Вложенное форматирование (`<b><i>текст</i></b>`)

✅ **HTML Preview** подсветит желтым если используешь неподдерживаемые теги!

### Типы сообщений и требования

| Тип | Требует | Optional | Особенности |
|-----|---------|----------|-------------|
| **text** | content_text | parse_mode | Могут быть кнопки |
| **photo** | content_file (file_id) | caption, parse_mode | Могут быть кнопки |
| **document** | content_file | caption, parse_mode | Могут быть кнопки |
| **video** | content_file | caption, parse_mode | Могут быть кнопки |
| **video_note** | content_file | — | **БЕЗ кнопок!** Круглое видео |
| **voice** | content_file | caption, parse_mode | **БЕЗ кнопок!** Голосовое сообщение |

### Задержки между сообщениями
```
delay_after_seconds = 0   → следующее сообщение тут же
delay_after_seconds = 5   → ждет 5 сек, потом отправляет
delay_after_seconds = 300 → ждет 5 мин

⚠️ Если > 600 сек (10 мин): создается фоновая задача (Celery)
   Бот может перезагрузиться, но продолжит отправлять вовремя
```

### Как получить File ID Telegram
```
Способ 1: Через бота
1. Отправь медиа-файл боту в Telegram
2. Включи логирование: logger.info(f"file_id: {message.photo[-1].file_id}")
3. Скопируй ID из логов

Способ 2: Forward из другого места
1. Отправь файл в чат
2. Forward-ни его боту
3. Получи file_id из message.photo[-1].file_id

❌ НЕ используй ссылки на файлы! Только file_id
```

## 🔘 Редактирование кнопок

### Как добавить кнопку
1. Нажми "Добавить" в секции "Кнопки шага"
2. Заполни:
   - **Текст**: то что видит пользователь на кнопке
   - **Тип**: URL / Callback / Payment
   - **Значение**: зависит от типа:
     - URL: полный адрес (`https://example.com`)
     - Callback: любая строка (обработается в обработчике)
     - Payment: ID товара для оплаты

### Порядок кнопок
- Drag & drop: перетащи кнопку на другую
- Или редактируй `button_order` напрямую (1, 2, 3...)

### Условия кнопок (продвинутое)
⚠️ **UI пока не реализован**, нужно через API или БД:
```json
{
  "require_tags": ["vip"],      // показать ТОЛЬКО если пользователь имеет тег
  "exclude_tags": ["inactive"]  // НЕ показывать если имеет тег
}
```

## ✅ Тестирование сообщений

### Перед отправкой пользователям ВСЕГДА тестируй!

1. **Установи свой Telegram ID в Settings**:
   - В админ панели: вкладка "Настройки"
   - Ключ: `admin_test_telegram_id`
   - Значение: твой Telegram ID (можно узнать у @userinfobot)

2. **Нажми "Отправить мне"**
   - Бот отправит ВСЕ сообщения шага точно как видит пользователь
   - **Включая кнопки!**
   - **С правильным форматированием!**
   - **С правильным parse_mode!**

3. **Проверь в Telegram**:
   - Открылись ли все сообщения в правильном порядке
   - Не ломаются ли теги HTML
   - Видны ли кнопки
   - Соответствуют ли задержки

❌ **Если не видишь кнопки в preview** → проверь что это не `video_note` или `voice`

## 📊 Полная логика отправки сообщения

### Когда пользователь входит в шаг:
```python
1. Загружаем все StepMessage для шага (отсортированы по message_order)
2. Загружаем все StepButton для шага (отсортированы по button_order)
3. Строим клавиатуру из кнопок (фильтруем по условиям пользователя)

4. Для каждого сообщения:
   a) Выбираем способ отправки в зависимости от message_type
   b) Применяем parse_mode к content_text или caption
   c) Прикрепляем клавиатуру (кроме video_note и voice)
   d) Отправляем
   
5. Если message.delay_after_seconds > 0:
   - ≤ 60 сек: await asyncio.sleep()
   - > 60 сек: создаем ScheduledTask с execute_at в будущем
   
6. Переходим к следующему сообщению

7. После ВСЕХ сообщений:
   - Шаг считается завершен
   - При клике на кнопку → переход на next_step_id
```

## 🐛 Часто встречающиеся ошибки

### ❌ "Тест-отправка не выполнена"
```
Причина: не установлен admin_test_telegram_id в Settings
Решение:
1. Вкладка "Настройки"
2. Найди "admin_test_telegram_id"
3. Введи свой Telegram ID (узнать у @userinfobot)
4. Сохрани (blur на поле)
```

### ❌ "Сообщение не отправилось пользователю"
```
Проверь:
1. message_type указан правильно (dropdown)
2. Для медиа: content_file заполнен корректным file_id
3. Если задержка > 600 сек: проверь что Celery/Redis работают
4. В логах: ищи "Failed to send message_id=..."
```

### ❌ "Кнопки не видны в боте"
```
Причины:
1. message_type = video_note (они не поддерживают кнопки!)
2. message_type = voice (они не поддерживают кнопки!)
3. Кнопки не добавлены к шагу
4. Все кнопки отключены (is_enabled=false) или не соответствуют условиям
```

### ❌ "Текст странно выглядит в Telegram"
```
Причина: использованы неподдерживаемые HTML теги
Решение:
1. Открой HTML Preview в админ панели
2. Если желтый border → неподдерживаемые теги
3. Используй только: <b>, <i>, <u>, <s>, <a>, <code>, <blockquote>
```

### ❌ "Задержка не работает"
```
Если delay > 600 сек:
- Нужно чтобы работал Celery и Redis
- Проверь переменные окружения: REDIS_DSN, CELERY_BROKER
- Проверь логи Celery в продакшене
```

## 🔐 Безопасность

### ✅ Что делает админ панель:
- Шифрует сохраняемые токены (bot_token, payment_api_key)
- Требует аутентификацию (ADMIN_PASSWORD_HASH)
- Сессии в памяти (SecureSession)

### ⚠️ Что НЕ делает (добавить в будущем):
- Audit logging (логирование кто что менял)
- Rate limiting на кнопку "отправить мне"
- Валидация на XSS в HTML редакторе
- Резервное копирование перед массовыми изменениями

## 🚀 Быстрый старт для новой воронки

### Сценарий: создать воронку "Пробный доступ на 7 дней"

```
1. Создай воронку
   - Вкладка "Воронки"
   - Кнопка "Новая воронка"
   - Имя: "Пробный доступ"

2. Создай шаги
   - Шаг 1 (message_order=1): приветствие
     - Сообщение 1: "Добро пожаловать! Вы получили пробный доступ на 7 дней"
     - Задержка: 0 сек
     - Кнопка 1: "Начать обучение" (callback: start_learning)
   
   - Шаг 2 (message_order=2): первый урок
     - Сообщение 1: "Урок 1: Основы" (с фото)
     - Задержка: 0 сек
     - Сообщение 2: "Вопрос: что такое X?"
     - Задержка: 5 сек (дать время подумать)
     - Кнопка 1: "Пропустить" (callback: skip)
     - Кнопка 2: "Мой ответ" (callback: submit_answer)

3. Тестируй ВСЕ шаги
   - Загрузи шаг 1
   - Нажми "Отправить мне"
   - Проверь в Telegram: правильный текст, фото, кнопки
   - Повтори для шага 2

4. Запусти воронку
   - Включи воронку (toggle "Активна")
   - Скопируй deeplink пользователям
   - Они автоматически войдут в воронку
```

## ✨ Лучшие практики

1. **Всегда тестируй перед отправкой пользователям**
   - Используй кнопку "Отправить мне"

2. **Используй HTML Preview для проверки форматирования**
   - Синее окно = все ОК
   - Желтое окно = неподдерживаемые теги

3. **Задержки между сообщениями**
   - Не более 1 сообщения без задержки (спам)
   - Естественное время: 3-5 сек между сообщениями

4. **Кнопки**
   - Максимум 3-4 кнопки (иначе не влезут)
   - Понятные текст (не "OK", а "Да, согласен")
   - Callback data < 64 символов

5. **Медиа-файлы**
   - Сначала отправь файл боту, получи file_id
   - Используй сразу, file_id может устареть
   - Проверь что изображение < 5MB

6. **Резервная копия**
   - Перед массовыми изменениями:
     ```bash
     pg_dump telegram_bot_db > backup_$(date +%Y%m%d).sql
     ```
   - При критической ошибке можно восстановить
